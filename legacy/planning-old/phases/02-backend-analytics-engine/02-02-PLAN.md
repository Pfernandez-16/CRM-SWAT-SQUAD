**calculateEmbudoGeneral_(currentLeads, previousLeads, allLeads, contactosIdx, interaccionesIdx, calificacionIdx, deals)**

Implements all 13 locked funnel metrics from CONTEXT.md. Each metric is a function passed to buildSegmentedMetricWithDelta_. Return an object with keys matching metric names:

Metric functions (each takes a leads subset and returns a count):
- **totalLeads**: `leads.length`
- **contactables**: Count leads where contactosIdx lookup has non-empty telefono_1 OR email (trim + check non-empty string)
- **contactados**: Count leads that have at least 1 entry in interaccionesIdx
- **conRespuesta**: Count leads that have at least 1 interaction with resultado === 'Contesto' (use exact string from data)
- **dialogoCompleto**: Count leads that have 2+ consecutive 'Contesto' toques. Logic: get all interactions for lead, filter to resultado='Contesto', sort by numero_toque ascending, check if any adjacent pair has toque numbers differing by exactly 1. A lead qualifies if it has at least one such consecutive pair.
- **dialogoIntermitente**: Count leads that have at least 1 'Contesto' interaction but do NOT qualify as dialogoCompleto. This is mutually exclusive with dialogoCompleto.
- **interes**: Count leads where calificacionIdx lookup has mostro_interes_genuino === 'Si' (check exact string from data)
- **descartados**: Count leads with status === 'Perdido'
- **asignadosVentas**: Count leads with status === 'Paso a Ventas'
- **carryOver**: Special metric -- from `allLeads` (not date-filtered), find leads where fecha_ingreso < dateIn AND status === 'Paso a Ventas' AND fecha_asignacion is between dateIn and dateOut. Use allLeads parameter, not the filtered subset.
- **montosInversion**: Sum of monto_proyeccion from deals linked to period leads (via id_lead match). Use buildSegmentedAmountWithDelta_.
- **dealsCerrados**: Count of deals linked to period leads where status_venta === 'Vendido'. Use buildSegmentedMetricWithDelta_.
- **montoCierres**: Sum of monto_cierre from deals linked to period leads where status_venta === 'Vendido'. Use buildSegmentedAmountWithDelta_.

For carryOver, the delta comparison uses the SAME logic but shifted: previous-carry-over = leads with fecha_ingreso < prevDateIn AND status='Paso a Ventas' AND fecha_asignacion between prevDateIn and prevDateOut.

IMPORTANT: For deal-based metrics (montosInversion, dealsCerrados, montoCierres), build a `currentLeadIds = new Set()` (or object with `var currentLeadIds = {}; for (var i = 0; i < leads.length; i++) { currentLeadIds[String(leads[i].id_lead)] = true; }`) of current period lead IDs, then filter deals where `deal.id_lead` is in that set. For segmentation of deal-based metrics, look up the LEAD's segment via calificacionIdx using the deal's id_lead (not the deal itself).

**Update getSDRReport()** to pass the correct parameters to calculateEmbudoGeneral_ (it was a stub with no params, now needs currentLeads, previousLeads, allLeads, contactosIdx, interaccionesIdx, calificacionIdx, deals).
