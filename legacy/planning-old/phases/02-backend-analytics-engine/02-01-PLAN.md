  <action>
Create gas-crm-project/Analytics.js with all the foundation infrastructure. The file structure should be:

**1. File header comment** explaining this is the analytics module, deployed alongside Codigo.js via clasp.

**2. Helper functions (private, suffix with _):**

- `filterByDateRange_(leads, dateIn, dateOut)` -- Filter leads array where fecha_ingreso is between dateIn and dateOut (inclusive). Parse dates with `new Date(ts).getTime()`. Handle NaN dates by excluding them. Return filtered array.

- `buildLeadInteractionsIdx_(interacciones)` -- Build a one-to-many index: `{ id_lead: [interaction1, interaction2, ...] }`. Return the map object. This is different from indexBy_ which is 1:1.

- `getSegment_(lead, calificacionIdx)` -- Given a lead and the calificacion index, return segment string. USES tipo_membresia FROM fact_calificacion (LOCKED decision from CONTEXT.md -- not servicio_interes). Logic: look up `calificacionIdx[String(lead.id_lead)]`, read its `tipo_membresia`. Return `'manufacturers'` if tipo_membresia is 'Manufacturers', `'individuals'` if 'Individuals', `'other'` for 'Attraction' or anything else, `'unknown'` if no calificacion or empty tipo_membresia. Always lowercase the return value.

- `calcPercentage_(part, total)` -- Return `(part / total * 100).toFixed(1)` as a string, or `'0.0'` if total is 0.

- `calcDelta_(currentVal, previousVal)` -- Return `((currentVal - previousVal) / previousVal * 100).toFixed(1)` as string. If previousVal is 0 and currentVal > 0, return `'100.0'`. If both are 0, return `'0.0'`.

- `buildSegmentedMetric_(leads, calificacionIdx, metricFn)` -- Apply metricFn to all leads (total), then to leads where getSegment_ returns 'manufacturers', then 'individuals'. Return object:
  ```javascript
  {
    total: { count: N, pct: '100.0' },
    manufacturers: { count: M, pct: calcPercentage_(M, N) },
    individuals: { count: I, pct: calcPercentage_(I, N) }
  }
  ```

- `buildSegmentedMetricWithDelta_(currentLeads, previousLeads, calificacionIdx, metricFn)` -- Same as buildSegmentedMetric_ but also runs metricFn on previousLeads for each segment, then adds `delta` field using calcDelta_ to each segment. Return:
  ```javascript
  {
    total: { count: N, pct: '100.0', delta: deltaStr },
    manufacturers: { count: M, pct: pctStr, delta: deltaStr },
    individuals: { count: I, pct: pctStr, delta: deltaStr }
  }
  ```

- `buildSegmentedAmountWithDelta_(currentLeads, previousLeads, calificacionIdx, amountFn)` -- Same pattern but amountFn returns a numeric sum (not a count). For amounts, `pct` is not meaningful so omit it. Return `{ total: { amount: X, delta: D }, manufacturers: { amount: Y, delta: D }, individuals: { amount: Z, delta: D } }`.

  - Build all indexes (contactosIdx via indexBy_, interaccionesIdx via buildLeadInteractionsIdx_, calificacionIdx via indexBy_ on 'id_lead')
