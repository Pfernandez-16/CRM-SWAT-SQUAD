Castillo & Company 02 — Scope Adicional ($150 extra)
Presupuesto Adicional

4 features acordados
como scope extra.

Total del proyecto: $350 USD ($200 base + $150 adicional).

A.1 — Deal Fichas Reestructuradas

Secciones Cotización y Cierre separadas visualmente, timestamps automáticos, multi-producto por deal, clasificación automática cross-sell/up-sell/venta directa.

A.2 — Cross-Selling Logic

Comparación automática del producto de interés del SDR vs productos de cierre del AE para clasificar tipo de transacción.

A.3 — AE Analytics / Reporte AE

Vista de reporte dedicada para Account Executives con métricas de deal pipeline, ranking AE por cierre, montos promedio y velocidad.

A.4 — Comparativa Personalizada

Tercer modo de comparación con fechas libres: cualquier rango vs cualquier rango. Carry-over incluido.

Castillo & Company 05 — Pendientes y Estado Técnico
Diferidos a Fase 2

Lo que no se incluyó
y por qué.

Funcionalidades que requieren infraestructura o procesos adicionales fuera del scope actual.

FeatureRazón
Importación masiva CSV/ExcelRequiere parseo de archivos, mapeo dinámico, validación masiva. Funcionalidad completa por sí sola.
Test 70/30 (split automático)Requiere lógica de distribución al ingreso, reglas por campaña, y dashboard de resultados.
BANT Score DecayNice-to-have: score disminuye conforme Timing se aleja. No bloquea operación.
Métrica SPAMNo hay columna fuente. El cliente necesita definir proceso operativo para marcar leads como Spam.
Correo con dominio propioLimitación de GAS: envía desde cuenta Google del usuario, no permite dominio personalizado.
Landing page parametrizableRequiere hosting separado o GAS Web App dedicada.
Estado técnico del sistema

Sistema en producción.

Codebase

~11,500 líneas de código en 5 archivos. Google Apps Script ES5 (backend) + Vue 3 Composition API (frontend).

Base de Datos

Star Schema v6 con 12 tablas normalizadas en Google Sheets. Escalable y limpia.

QA

4 bugs críticos encontrados y corregidos (handoff routing, SDR CVR, duplicados, status canónico). Verificación end-to-end completada.

Deploy

Sistema en producción via Google Apps Script. Deploy con clasp push.
