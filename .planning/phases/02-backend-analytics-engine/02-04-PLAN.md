**calculateRazonesNoPasoVentas_(currentLeads, previousLeads, calificacionIdx)**

This section answers "Why didn't leads pass to Sales?" Since fact_leads has no razon_perdida field, we derive reasons from fact_calificacion BANT fields for leads with status = 'Perdido'.

Logic:
1. Define reason categories as an array of objects, each with a `label` and a `testFn`:
   ```javascript
   var reasons = [
     { key: 'noPerfilAdecuado', label: 'No perfil adecuado', testFn: function(lead, calif) { return calif && String(calif.perfil_adecuado) === 'No'; } },
     { key: 'sinPresupuesto', label: 'Sin presupuesto', testFn: function(lead, calif) { return calif && String(calif.tiene_presupuesto) === 'No'; } },
     { key: 'sinInteresGenuino', label: 'Sin interes genuino', testFn: function(lead, calif) { return calif && String(calif.mostro_interes_genuino) === 'No'; } },
     { key: 'necesitaTercero', label: 'Necesita tercero para decidir', testFn: function(lead, calif) { return calif && String(calif.necesita_decision_tercero) === 'Si'; } },
     { key: 'noEntendioMarketing', label: 'No entendio info marketing', testFn: function(lead, calif) { return calif && String(calif.entendio_info_marketing) === 'No'; } },
     { key: 'otros', label: 'Otros / Sin calificar', testFn: null }  // Catch-all
   ];
   ```
2. Filter current leads to only those with status === 'Perdido'
3. For each reason (except 'otros'), count leads matching that testFn. A lead can match multiple reasons.
4. For 'otros': count leads with status 'Perdido' that matched NONE of the other reason testFns AND either have no calificacion record or have no 'No' answers.
5. Apply segmentation and delta using the established pattern. For each reason key, call buildSegmentedMetricWithDelta_ where the metricFn filters to status='Perdido' leads matching the testFn.

Return format:
```javascript
{
  noPerfilAdecuado: { total: {count, pct, delta}, manufacturers: {...}, individuals: {...} },
  sinPresupuesto: { total: {count, pct, delta}, manufacturers: {...}, individuals: {...} },
  sinInteresGenuino: { ... },
  necesitaTercero: { ... },
  noEntendioMarketing: { ... },
  otros: { ... },
  _totalDescartados: { total: {count, pct, delta}, manufacturers: {...}, individuals: {...} }
}
```
Include `_totalDescartados` as a convenience total (all leads with status 'Perdido').

**calculateRazonesPerdioVenta_(currentLeads, previousLeads, calificacionIdx, deals)**

This section answers "Why was the sale lost?" Uses fact_deals.razon_perdida directly.

Logic:
1. Build current period lead ID set from currentLeads: `var currentLeadIds = {}; for (var i = 0; i < currentLeads.length; i++) { currentLeadIds[String(currentLeads[i].id_lead)] = true; }`
2. Filter deals where `currentLeadIds[String(deal.id_lead)] === true` AND `deal.status_venta === 'Perdido'`
3. Get the list of all possible razon_perdida values from cat_opciones (hardcode the 13 known values)
4. For each reason, count deals matching that razon_perdida value. For segmentation, look up the LEAD's segment (via calificacionIdx using deal.id_lead).
5. Also count deals with empty/null razon_perdida as 'Sin especificar'
6. Use buildSegmentedMetricWithDelta_ for each reason. The metricFn receives a leads subset; inside it, build leadIds set from that subset, filter deals where id_lead is in that set AND status_venta is 'Perdido' AND razon_perdida matches the reason.
